gcc main.c parse_args.c push_swap_utils.c erro_handler.c utils.c -o push_swap


1. Reestruturação da Arquitetura Base (O(1))
Antes: O programa utilizava uma lista ligada simples onde cada inserção exigia percorrer todos os nós
(while (last->next != NULL)), gerando uma complexidade ineficiente de O(n^2)no preenchimento da stack.
Hoje: Mudámos para uma Estrutura Encapsuladora (t_stack) que armazena os ponteiros diretos para o topo (top), fundo (bottom) e o tamanho atual (size). A inserção de dados foi refatorada na função add_bottom, passando a rodar em tempo constante O(1), eliminando por completo os loops de busca
2. Módulo de Entrada de Dados e Captura de Flags (parse_args.c)Implementação do Gestor de Configuração: Desenhou-se a função parse_flags para ler o argv e configurar a estrutura t_config sem o uso de variáveis globais (proibidas pela norma). O programa agora deteta dinamicamente o modo de benchmark (--bench) e as 4 estratégias obrigatórias (--simple, --medium, --complex, --adaptive).Sincronização do Fluxo: O parser devolve o índice exato (start_index) onde as flags terminam, permitindo que o main ignore as opções e processe apenas os números da stack
3. Caça aos Bugs Críticos (Depuração de Memória)Durante os testes de integração, detetámos e corrigimos 4 falhas graves que causavam comportamentos indefinidos ou crashes no terminal:Correção de Sinais Inclusivos no ft_atoi_safe: O loop que convertia os caracteres excluía o '0' e o '9' devido aos operadores < e >. Atualizámos para >= e <= para garantir que números contendo ou terminados em 0 e 9 (como o teu teste com o 9) sejam convertidos corretamente.Correção Lógica no is_numeric: A função usava o operador && (if (str[i] < '0' && str[i] > '9')), uma condição matematicamente impossível que validava letras como se fossem números. Corrigido para || (menor que '0' OU maior que '9').
Resolução de Desreferenciação Nula: Removeu-se a linha result = 0; que reinicializava o ponteiro de destino como NULL dentro do ft_atoi_safe, o que originava um crash imediato ao tentar escrever o valor final.Resolução do Segmentation Fault na Inicialização: Na função init_stack_a, o programa tentava passar o endereço do ponteiro (&stack_a), gerando um tipo t_stack  incompatível. Ajustámos a assinatura para passar apenas stack_a, estabilizando o mapeamento de memória.4. Gestão de Erros Blindada
(erro_handler.c)Implementou-se a função free_stack que limpa todos os nós alocados dinamicamente no Heap de forma iterativa em caso de falha a meio do parsing.Garantiu-se que qualquer input inválido (duplicados, overflows detetados pelo ft_atoi_safe ou caracteres estranhos capturados pelo is_numeric) resulte na impressão estrita de "Error\n" no canal de erros padrão (stderr, descritor 2), seguida de um exit(1) limpo e sem memory leaks
